# china-minsk.by — доставка товаров из Китая в Беларусь под ключ

**Сайт:** [https://china-minsk.by](https://china-minsk.by)
**Тип:** Промо-сайт логистической компании с генерацией лидов
**Год:** 2025–2026

## О проекте

Полностью самописный сайт логистической компании, занимающейся доставкой
товаров из Китая в Беларусь: карго, официальная доставка, таможенное
оформление, выкуп с 1688 и Taobao, консолидация грузов.

Цель — собирать заявки на расчёт стоимости и максимально широко
индексироваться в поиске по геозапросам по всем крупным городам Беларуси.
Никаких CMS, конструкторов и шаблонов — статика, заточенная под скорость
загрузки и SEO.

## Стек

| Слой | Технология |
|---|---|
| Frontend | Pure HTML5 / CSS3 / Vanilla JS (без фреймворков и сборщиков) |
| Шрифты и иконки | Google Fonts (Noto Sans, Playfair Display), Font Awesome |
| Анимации | IntersectionObserver API, CSS keyframes |
| Backend для лидов | Python 3, FastAPI, Pydantic, httpx |
| Контейнеризация | Docker + docker-compose |
| Reverse proxy | nginx |
| Уведомления | Telegram Bot API |
| Хостинг сайта | Shared cPanel, Беларусь |
| Хостинг API | Отдельный VPS (`api.china-minsk.by`) |

## Структура сайта

- **Главная** — лид-форма, услуги, география, преимущества, кейсы
- **16 городских лендингов** — Минск, Брест, Витебск, Гомель, Гродно,
  Могилёв, Барановичи, Бобруйск, Лида, Молодечно, Новополоцк, Орша,
  Пинск, Полоцк, Солигорск, Жлобин. Каждый — отдельная SEO-страница
  с уникальной OG-картинкой, метаданными и геолокальным контентом.
- **7 страниц услуг** — карго, официальная доставка, таможенное
  оформление, сертификация, выкуп с маркетплейсов, консолидация, о компании
- **404** — кастомная страница

Итого ~25 страниц, каждая с собственным `<title>`, описанием,
canonical, Open Graph и Schema.org разметкой.

## Лид-форма

Двумодальная: «Есть ссылка на товар» / «Опишу товар словами» —
поля адаптируются под выбранный сценарий. Полная клиентская валидация:

- URL-валидация для ссылки на товар
- Маска и валидация белорусского телефона `+375 (XX) XXX-XX-XX`
- Email-валидация
- Город из выпадающего списка с привязкой к городским страницам

Заявки уходят на FastAPI-эндпоинт `api.china-minsk.by/api/notify`,
оттуда — в Telegram-чат менеджеров. Подтверждение пользователю
показывается inline, без перезагрузки.

## Технические решения

### SEO и индексация

- **Schema.org LocalBusiness** JSON-LD с указанием телефона, адреса,
  обслуживаемых городов, контактных точек, ссылок на Viber/Telegram
- **Open Graph + Twitter Card** на каждой странице, с уникальной
  превью-картинкой 1200×630 (24 OG-изображения)
- Корректные `canonical`, `robots`, `sitemap.xml`, `meta description`,
  семантические `<h1>`–`<h3>`
- Preconnect к Google Fonts и Font Awesome для ускорения первой отрисовки
- Lazy-loading и явный `width`/`height` у изображений — без layout shift

### Производительность

- Чистый HTML без рантайма-фреймворков → быстрый Time to Interactive
- Адаптивные изображения через `srcset` + `sizes`
- WebP в основной графике
- CSS встроен в `<head>` критичной частью, остальное — отложено
- Минимум сторонних скриптов, никакой аналитики-тяжеловесов

### UX

- Адаптив mobile-first, основной CTA продублирован в шапке, hero
  и плавающей кнопке
- Поп-апы для лид-формы и контактов с блокировкой скролла фона
  и закрытием по Escape / клику на overlay
- IntersectionObserver-анимации появления секций при прокрутке
- Поэтапная подсветка карточек городов с задержкой

### Инфраструктура

- Статика заливается через `scp` на cPanel-хостинг по выделенной
  схеме деплоя (отдельная папка addon-домена, изолированно от
  основного сайта)
- API-сервис в Docker на отдельном VPS, с CORS строго на
  `china-minsk.by`, валидацией входящих данных через Pydantic
  и rate-friendly архитектурой
- Telegram-уведомления с ретраями через httpx
