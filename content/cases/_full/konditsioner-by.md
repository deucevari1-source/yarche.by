# кондиционер.by — каталог и установка кондиционеров в Минске

**Сайт:** [https://konditsioner.by](https://konditsioner.by)
**Тип:** Промо-сайт услуг установки с автогенерируемым каталогом и лидгеном
**Год:** 2026

## О проекте

Самописный сайт сервиса по установке бытовых кондиционеров в Минске:
89 моделей сплит-систем с реальными ТТХ и фотографиями, калькулятор
стоимости под ключ, лид-форма с уведомлением в Telegram и большая
SEO-сетка из 70+ статических страниц по районам Минска, типам помещений,
брендам и сравнениям.

Главная инженерная задача — автоматически тянуть каталог из xlsx-прайса
дистрибьютора (ООО «Биоконд», эксклюзивный поставщик MDV и TCL в РБ):
с реальными характеристиками, фото серий из встроенных в Excel картинок,
группировкой по сериям, обновлением без ручного ввода.

В мае 2026 — ребрендинг и миграция домена `air20.by` → `konditsioner.by`
(URL на латинице, видимый бренд `кондиционер.by` кириллицей).

## Стек

| Слой | Технология |
|---|---|
| Frontend | Next.js 14 (App Router), React 18, TypeScript strict, Tailwind CSS 3 |
| Рендеринг | SSG с `generateStaticParams` для всего каталога и SEO-сетки |
| Backend для лидов | Next.js Route Handlers, Telegram Bot API |
| Хранилище данных | JSON-файлы в репо (`data/*.json`) — без базы |
| Картинки | next/image + WebP, кастомный конвертер PNG → WebP с ресайзом |
| Парсер прайса | Python 3 + openpyxl + Pillow |
| Контейнеризация | без Docker, прямой Node-процесс под PM2 |
| Reverse proxy | nginx + Let's Encrypt |
| Хостинг | VPS, Беларусь |
| Деплой | `git pull` + `npm run build` + `pm2 restart` |

## Структура сайта

- **Главная** — герой с калькулятором стоимости, шоукейс «3 варианта
  под комнату» (эконом/оптимум/премиум), блок «Как это работает»,
  социальное доказательство, FAQ, лид-форма
- **/catalog** — каталог 89 моделей с фильтрами по бренду, мощности,
  типу (инвертор / on-off), наличию и сортировкой по цене/мощности
- **/catalog/[slug]** — карточка модели: реальное превью из прайса,
  цена под ключ, табличный блок «Технические характеристики», список
  «Особенности серии» (до 16 пунктов из описания серии в прайсе)
- **/catalog/brand/[brand]** — 6 бренд-лендингов с SEO-описанием,
  сильными/слабыми сторонами, сравнительной таблицей и моделями
- **/minsk/[district]** — 23 страницы под все 9 районов и 14 микрорайонов
  Минска
- **/tasks/[slug]** — 10 страниц под типовые задачи: квартира, спальня,
  детская, офис, кухня, новостройка, частный дом, после ремонта,
  срочно, монтаж в 2 этапа
- **/blog/[slug]** — 15 SEO-статей (5 написаны полностью, 10 — заглушки
  под расширение контента)
- **/compare/[pair]** — 4 страницы сравнения: Daikin vs Mitsubishi,
  Gree vs Haier, Инвертор vs On/Off, Сплит vs Мультисплит
- **/calculator, /services, /about, /contacts, /policy, /oferta** —
  служебные

Итого ~100 пререндеренных страниц, каждая со своим `<title>`,
`description`, canonical, Open Graph и JSON-LD.

## Парсер прайса дистрибьютора

Главная инженерная фича. Прайс приходит в xlsx (~20 MB, ~1000 строк
на лист, фото моделей встроены прямо в ячейки как Excel-картинки).

- **openpyxl + ручная разборка `xl/drawings/*.xml`** для привязки
  встроенных фото к строкам моделей через row anchors
- Группировка моделей по серии: парсер сам распознаёт заголовки серий
  («Сплит-системы серии INFINI Inverter…»), собирает буфер из
  checkmark-фич следующей строки и атачит к моделям до следующего
  заголовка
- Извлекаются: мощность охлаждения/обогрева в кВт, цена, наличие,
  габариты и вес внутреннего/наружного блока, тип фреона (R32/R410A),
  питание, диаметр труб, класс энергоэффективности, Wi-Fi, гарантия,
  имя серии, до 16 особенностей серии
- Цена под ключ = РРЦ + базовая стоимость монтажа (берётся из прайса
  калькулятора `data/pricing.json`)
- Идемпотентность: повторный запуск обновит существующие записи
  по коду модели, не задвоит. Сохраняет `isHit` и кастомные правки.
- Раздельный скрипт `fix_novinka_images.py` для случаев, когда в xlsx
  у новых SKU вместо фото вставлен штамп «НОВИНКА» — подменяет
  на реальные фото серий, скачанные с `mdv-aircond.ru` и `tcl-shop.by`

## Технические решения

### Каталог: клиентские фильтры поверх SSG

- Страница `/catalog` — серверный компонент, отдаёт SEO-контент и все
  89 моделей разом в client-боундари
- `<CatalogGrid/>` — клиентский компонент с состоянием:
  - **Sticky top-bar** (липкий при скролле): дропдаун сортировки,
    кнопка «Фильтры» с бейджем количества активных, счётчик
    «Найдено X / Y»
  - **Чипы активных фильтров** с крестиком для удаления по одному
  - **Сворачиваемая панель**: класс (segmented), бренд (пилюли),
    мощность BTU (пилюли с tooltip-ом в кВт), «только инвертор»,
    «только в наличии»
  - Сортировка: по умолчанию (Оптимум → Премиум → Эконом, внутри
    по цене), цена ↑/↓, мощность ↑/↓
  - При сортировке «по умолчанию» — модели сгруппированы секциями
    с h2-заголовками классов, при остальных — единым списком

### Сжатие фото

После публикации первой версии заметили медленную загрузку картинок
на проде. Исходные PNG из xlsx были 100-3000 KB при отображении максимум
220 px шириной. Скрипт `optimize_images.py` (Pillow):
- Конвертирует PNG → WebP с ресайзом до 800 px max
- Quality 85, method=6 для максимальной компрессии
- Обновляет ссылки в `data/models.json`
- Удаляет исходники

Результат: 8.9 MB → 200 KB (≈45× экономии), время загрузки
картинок упало в 10+ раз.

### SEO

- **JSON-LD** на разных типах страниц:
  - Главная: `LocalBusiness` + `Organization` + `WebSite`
  - Модели каталога: `Product` + `BreadcrumbList` + `AggregateRating`
  - Услуги: `ItemList` (`Service`)
  - Районы: `BreadcrumbList` + `Service` + `FAQPage`
  - Задачи: `BreadcrumbList` + `Service`
  - Статьи: `Article` + `BreadcrumbList`
  - Бренды: `BreadcrumbList` + `ItemList`
  - Сравнения: `BreadcrumbList` + `Article`
- **Динамический `sitemap.xml`** через `prebuild`-скрипт на Node
  (`scripts/generate-sitemap.mjs`) — генерируется при каждой сборке
  и включает все ~100 URL
- Canonical, Open Graph, Twitter Card, корректный `robots.txt`
- На уровне маршрутов: `dynamicParams = false` для каталога —
  скрытые/удалённые модели сразу дают 404, без серых страниц

### Лид-форма и калькулятор

- Калькулятор тарифицируется по чистой логике в `lib/calc.ts`:
  ступенчатый прайс по площади + класс + тип стены (кирпич/панель/монолит)
  + материалы
- Опция «своё оборудование» — пересчёт только за монтаж + фикс. наценка
- Один и тот же `calculate()` используется и на клиенте в калькуляторе,
  и на сервере в API-хендлере заявки — гарантированно сходящиеся
  цифры
- Заявка → POST `/api/lead` → Telegram-уведомление через Bot API
- Fallback: если переменные `TELEGRAM_BOT_TOKEN`/`CHAT_ID` не заданы,
  заявка пишется в лог сервера + резервную папку — не теряется

### Ребрендинг с миграцией домена

`air20.by` → `konditsioner.by` в мае 2026. Сохранены SEO-сигналы:
- URL/email/технические идентификаторы — латиница (`konditsioner`)
- Видимый бренд в текстах и логотипе — кириллица (`кондиционер.by`)
- 301-редирект со всех старых URL на новые
- `Change of Address` в Google Search Console и Я.Вебмастере
- Переименованы внутренние неймспейсы: sessionStorage, localStorage,
  CustomEvent, package.json — везде `konditsioner` вместо `air20`

### Скрытые модели

Каталог поддерживает флаг `hidden: true` на уровне модели. Скрытые:
- Не попадают в `getAllModels()` и в `generateStaticParams`
- Не пререндерятся, дают 404 при прямом обращении
- Данные сохраняются в `models.json` — можно вернуть, поставив `false`

Это используется для seed-моделей, у которых пока нет реальных данных
(бренды без активного импорта), и для моделей, выведенных из ассортимента.

## Деплой

- VPS, Node 20, PM2, nginx, Let's Encrypt
- Деплой одной серией команд после `git push`:
  ```bash
  cd /var/www/konditsioner.by
  git fetch origin main
  git reset --hard origin/main
  npm run build          # prebuild сам пересоберёт sitemap.xml
  pm2 restart konditsioner
  ```
- `.env.local` с секретами (Telegram-токен, чат, site URL) хранится
  только на сервере, в `.gitignore`
- `npm ci` запускается только если менялся `package-lock.json` —
  быстрые итеративные деплои за 15-20 секунд
