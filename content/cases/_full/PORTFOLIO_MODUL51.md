# Модуль 51

**Лидген-сайт с трёхмерной сценой, которая живёт за контентом всех страниц.**
Один WebGL-канвас остаётся примонтированным при навигации между разделами:
3D-модель сборного блока плавно меняет вид и материалы в зависимости от
маршрута — никакого «прыжка» между фоном-картинкой и реальным
рендером, как у большинства промо-сайтов в этом сегменте.

## Контекст

Компания поставляет сборные модули фиксированной длины (6 м) двух
типоразмеров (ширина 3.0 и 2.5 м) и адаптирует их под задачу клиента:
бытовка, КПП, торговая точка, дача, баня, склад, многомодульный жилой
блок. До рестайла позиционировалась как «контейнерная» компания. Задача
агентства — перевести коммуникацию в B2B-формат «сборные модули под
задачу», а не «дешёвые контейнеры».

## Что сделано

### 1. Живая 3D-сцена как фон

- Единый `<canvas>` в iframe (`/scene.html`), который **не размонтируется
  при навигации**. React-хост шлёт в iframe `postMessage` с режимом и
  целевой камерой; сцена плавно переключает материалы и видимость
  подгрупп.
- 10 режимов сцены: контейнер, КПП, банный модуль с трубой и купелью,
  дачный домик, мини-маркет с прилавком, фуд-поинт с маркизой и
  неоновой вывеской, четыре-модульный жилой блок «квадратом», склад
  с торцевыми распашными дверями, доставка краном-манипулятором,
  «о компании» с тремя бригадными фигурами.
- На каждой странице — свой пресет камеры с 800 мс easeInOut-переходом
  между ракурсами.
- Лёгкий sway-эффект в холостом режиме (камера медленно «дышит»),
  пауза при пользовательском интеракшене.

### 2. Каталог с переключателем типоразмеров

- 5 категорий, ~8 продуктов с заранее структурированной моделью
  (`Product`, `ProductFormat`, `ProductOption`) под будущий конфигуратор.
- На карточке — клиентский компонент **`<FormatSwitcher/>`** с двумя
  таблетками (6×3.0 / 6×2.5 м). Клик мгновенно меняет габариты и
  площадь, не уходя в подзагрузку. Размер в карточке стоит «крупно
  в шапке» — это главный коммерческий параметр.

### 3. Маска белорусского телефона

Свой компонент `<PhoneInput/>` поверх `react-hook-form`:
- Принимает любой ввод (`+375…`, `8029…`, чистые цифры, вставку из
  буфера) и нормализует в `+375 (XX) XXX-XX-XX`.
- Курсор фиксируется в конце поля, чтобы пользователь не правил
  префикс случайно.
- Zod-валидация: regex точно на маску — нельзя отправить полузаполненный
  номер.

### 4. Брендинг и подпись агентства

- Чёрный «чип» с жёлтым логотипом **YARCHE** в подвале (контраст
  на светлом фоне сайта-клиента; шрифт Unbounded 800 через
  `next/font/google`).
- Лого-«пасхалка» на задней стене 3D-модели контейнера: «YARCHE.by»
  жёлтым с мягким glow и чёрным `.by` под ним. Видна только если
  пользователь покрутит сцену орбитой.
- Логотип запечён в `/public/yarche-logo.png` через одноразовую
  страницу-генератор `_bake-yarche.html`, чтобы в iframe не таскать
  второй экземпляр шрифта (~50KB экономии на каждом первом визите).

## Технические решения, на которые стоит обратить внимание

- **iframe-изоляция Three.js.** Сцена живёт в отдельном документе,
  поэтому HMR Next-приложения не убивает WebGL-контекст и текстуры
  при правках. Перерендер сцены — только при бампе явного
  `cache-buster`-параметра (`?v=NN`).
- **Динамический `setViewOffset`** камеры: на десктопе модель сдвинута
  в правую треть канваса (под контент-колонку слева), на мобиле —
  в верхнюю часть (под скролл текстом снизу).
- **SSG + `generateStaticParams`** для всех страниц каталога. Сцена и
  лид-форма — клиентские; остальное статически пререндерится.
- **TypeScript + zod schema-first** в лид-форме: одна схема даёт и
  валидацию, и тип `LeadValues` для react-hook-form.
- **Tailwind v4** с CSS-переменными темы (`--color-paper`, `--color-ink`)
  для лёгкого свапа палитры — изначально сайт был в «editorial»-стиле
  с Fraunces serif, потом полностью переброшен в B2B-функциональный
  Inter за один день.
- **Сабмодульная архитектура репозитория**: код приложения живёт в
  отдельном репо `containers_web`, родительский репо хранит указатель
  на нужный коммит — удобно для версионирования деплоев независимо от
  обвязки проекта.

## Стек

| Слой | Что |
|---|---|
| Фреймворк | **Next.js 16** (App Router) + **React 19** |
| Язык | TypeScript |
| 3D | **Three.js r128** (vanilla, без R3F) — нужен был полный контроль над сценой и iframe-изоляцией |
| Формы | react-hook-form + zod + @hookform/resolvers |
| Стили | Tailwind v4, CSS variables для темы |
| Шрифты | Inter (основной), Onest + Unbounded (брендинг агентства) — все через `next/font/google` |
| Хостинг | Vercel/Netlify, auto-deploy с `main` |

## Результат

- Сайт развёрнут, лидген-форма работает, каталог наполнен 8 SKU.
- Среднее время до первой осмысленной интеракции — низкое: 3D-сцена
  стартует параллельно с DOM, контент колонки готов до того, как
  сцена закончит инициализацию.
- Бренд клиента перепозиционирован: «контейнеры» убраны из всей копии,
  компания читается как поставщик/интегратор сборных модулей.
