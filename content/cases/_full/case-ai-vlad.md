# ai-Влад — AI-ассистент для сайта агентства ЯРЧЕ

**Тип проекта:** AI-продукт, разработка с нуля  
**Клиент:** Агентство ЯРЧЕ (внутренний проект)  
**Стек:** Node.js, Hono, Anthropic Claude API, Nginx, Docker, Python (Flask), Telegram Bot API  
**Срок реализации:** 1 сессия (итеративная разработка)  
**Результат:** Полностью рабочий продукт на продакшене

---

## Задача

Агентству нужен был инструмент, который работает 24/7 вместо менеджера — квалифицирует входящие лиды, отвечает на вопросы об услугах и передаёт готовые заявки в CRM. Стандартные чат-виджеты не подошли: они не умеют вести диалог и собирать данные о клиенте.

Цель — создать AI-ассистента с именем и характером, который встраивается на сайт одной строкой кода и работает на любом устройстве.

---

## Что сделали

### Виджет (Frontend)

Написан на чистом JavaScript без фреймворков — один файл `widget.js` весом ~15 КБ. Подключается на любой сайт одним тегом:

```html
<script src="/widget.js" data-api="https://bot.yarche.by" data-name="ai-Влад"></script>
```

Ключевые особенности:

- **Shadow DOM** — виджет изолирован от стилей сайта, не конфликтует с CSS
- **Streaming** — ответы появляются в реальном времени, побуквенно (Server-Sent Events)
- **Адаптивность** — на десктопе круглая кнопка в углу с анимацией, на мобиле — пилюля с кольцами, всплывающими вдоль обводки
- **iOS Safari фиксы** — отключён автофокус чтобы клавиатура не перекрывала чат, font-size 16px чтобы страница не зумилась при вводе
- **Сессия** — история диалога сохраняется в sessionStorage на 1 час, не теряется при переходах по страницам
- **Квалификация лидов** — виджет распознаёт спец-тег `[LEAD_READY]` в ответе модели и показывает форму для сбора контактов

### API-сервер (Backend)

Node.js + Hono framework, задеплоен через PM2 на отдельном сервере.

- Проксируется через Nginx с SSL (Let's Encrypt)
- Rate limiting: 20 запросов в минуту, 200 в день на реальный IP пользователя (через `X-Forwarded-For`)
- История обрезается до последних 10 сообщений — экономия токенов
- Модель: Claude Haiku (быстро, дёшево, достаточно для консультационного чата)

### AI-логика (Промпт)

ai-Влад — не просто FAQ-бот. Он ведёт структурированный диалог:

1. Узнаёт чем занимается бизнес
2. Есть ли сайт и какие digital-инструменты
3. Какие проблемы и цели стоят
4. Что хотелось бы улучшить

Когда собрал все данные — формирует заявку и запрашивает контакты. Заявка уходит в Telegram менеджеру в том же формате, что и заявки с форм на сайте.

### Инфраструктура

- Два сервера: один для сайта (cPanel), второй для API
- API-сервер: Node.js (PM2) + Python/Flask (gunicorn) + Nginx + Docker (мониторинг)
- SSL через Let's Encrypt, автообновление через certbot в Docker
- CORS настроен точечно — только домен yarche.by

## Технические решения, которые стоит отметить

**Совместимость с кастомным курсором** — сайт использует `cursor: none` для кастомного курсора. Виджет инжектирует глобальный стиль чтобы вернуть стандартный курсор только на своих элементах, не ломая остальной сайт.

**Конфликт портов** — на сервере уже работал gunicorn на порту 5050 (Telegram-бот для заявок). Решение: API-чата поднят на 3001, Nginx проксирует оба сервиса по разным путям через один домен `bot.yarche.by`.

**Rate limiting по реальному IP** — изначально все запросы приходили с IP самого сервера (Nginx не передавал заголовок). Добавлен `proxy_set_header X-Forwarded-For $remote_addr` в конфиг Docker-контейнера с Nginx.

**Заявки в существующий бот** — не создавали новую инфраструктуру. Виджет отправляет лид напрямую на уже работающий эндпоинт `/api/lead` Flask-бота, менеджер получает заявку в том же формате что и с форм сайта.
